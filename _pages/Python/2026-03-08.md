파이썬에서 `type` 같은 메타클래스(metaclass)는 "클래스를 만드는 클래스"이다.   
이게 무슨 말이냐면, 파이썬에서 모든 것은 객체이다.
`int`, `str` 모든 것이 다 객체이다.   
심지어 class까지 객체이다.

{% highlight python %}
x = 3             # <class 'int'>
s = "hello"         # <class 'str'>

def greet():
    print("hi")     # <class 'function'>
{% endhighlight %}

그렇다면 class는 어떤 class로 부터 생성된 객체일까?   
그때 사용되는 것이 메타클래스인 `type`이 된다.   
즉, `type`은 **클래스를 만드는 클래스** 또는 **파이썬의 기본 메타클래스**라고 말할 수 있다.


아래 코드를 봐보자.
{% highlight python %}
# 모든 클래스는 type의 인스턴스
class myClass(object):
    x = 10
    
         @classmethod        # 인스턴스를 만들지 않고 클래스 자체에서 바로 계산하고 싶을 경우..
    def sum(self, y):
        print(self.x + y)

# classmethod를 사용하여 클래스 자체에서 바로 sum() 호출
myClass.sum(5)
obj = myClass()
obj.sum(100)


# type 클래스로 클래스 myClass2를 생성
myClass2 = type('myClass2', (object,), {'x':10, 'sum':lambda self, y: print(self.x + y)})
obj2 = myClass2()
print(obj2.x)
obj2.sum(30)

'''
실행결과
15
110
10
40
'''
{% endhighlight %}

보다시피, `type`으로 클래스를 정의하는 것을 볼 수 있다.   
이는 `type`이 클래스를 생성하는 클래스임을 보여준다.
