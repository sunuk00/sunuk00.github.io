# `in` 연산자

파이썬에서 `in`과 `not in` 연산자는 membership operator이다.   
이는 컨테이너 안에 값이 존재하는지 여부를 확인하는 연산자다.

내부적으로 `in`은 `__contains__`라는 특수 메서드를 호출한다.   

만약 `__contains__`가 없으면 파이썬은 iteration이 가능한 객체인지 확인한다.
즉, 이 경우에는 `__iter__`이 있는지 찾는다.
만약 있다면, 요소를 하나씩 꺼내면서 비교를 해본다.

그런데 만약 `__iter__`도 없는 경우, `__getitem__` 메소드를 호출하여 인덱스로 접근이 가능한지 확인한다.

만약 세가지가 다 정의가 안되어 있다면 파이썬은 membership test를 할 방법이 없어서, `TypeError` 예외가 발생한다.

{% highlight python %}
class MyClass:
    def __init__(self):
        self.items = ['apple', 'banana', '수박']

    def __contains__(self, item):
        if item == '티라노사우르스':
            return True
        return item in self.items
    
c = MyClass()
print('티라노사우르스' in c) # True
print('apple' in c)        # True
print('watermelon' in c)   # False
{% endhighlight %}

위의 코드를 보면 `MyClass` 클래스에 `__contains__` 메소드를 정의하여
`in` 연산자를 내가 다시 정의했다.

# `__contains__`와 `__iter__`의 성능 차이

list나 tuple의 `__contains__`는 내부적으로 루프를 돌기 때문에 $$O(N)$$이지만,
dictionary나 set의 경우 `__contains__`는 해시 테이블을 사용하기 때문에 평균적으로 $$O(1)$$이 걸린다.

그러나 `__iter__`을 사용하게 되면 선형적으로 확인을 하기 때문에 반드시 $$O(N)$$이 소요된다.
즉, `__contains__`를 사용하면 해당 자료구조의 최적화된 방식으로 검사할 수 있어서 효율적이다.
