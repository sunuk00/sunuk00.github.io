[U-Net: Convolutional Networks for Biomedical Image Segmentation](https://arxiv.org/abs/1505.04597) <br>

> While convolutional networks have already existed for a long time [8], their success was limited due to the size of the available training sets and the size of the considered networks. The breakthrough by Krizhevsky et al. [7] was due to supervised training of a large network with 8 layers and millions of parameters on the ImageNet dataset with 1 million training images. Since then, even larger and deeper networks have been trained [12].

여기서 말하는 Krizhevsky et al. [7]는 AlexNet을 말하며, ImageNet 데이터셋을 이용하여 8개의 레이어와 수백만 개의 파라미터를 가진 대규모 네트워크를 학습시킨 것을 의미한다. 이로 인해 CNN이 이미지 분류 작업에서 큰 성공을 거두게 되었다.

> The typical use of convolutional networks is on classification tasks, where the output to an image is a single class label. However, in many visual tasks, especially in biomedical image processing, the desired output should include localization, i.e., a class label is supposed to be assigned to each pixel. Moreover, thousands of training images are usually beyond reach in biomedical tasks.

localization이란 이미지 내에서 객체의 위치를 식별하는 것을 의미하며, 이는 각 픽셀에 클래스 레이블을 할당하는 것이다.

그런데 biomedical tasks 같은 경우가 이러한 localization이 중요한데, 애초에 학습 데이터를 얻는 것이 어려움 ㅠ.ㅠ

> Hence, Ciresan et al. [1] trained a network in a sliding-window setup to predict the class label of each pixel by providing a local region (patch) around that pixel as input. First, this network can localize. Secondly, the training data in terms of patches is much larger than the number of training images.

그림을 보면 알겠지만, 이미지를 패치 단위로 나누고, 각 패치를 CNN에 입력으로 넣고, 각 패치의 중심 픽셀에 대한 클래스 레이블을 예측하는 방식이다. 즉, 각 패치의 중심 픽셀에 대한 클래스 레이블을 예측하는 것이므로, localization이 가능하다.

> Obviously, the strategy in Ciresanet al. [1] has two drawbacks. First, it is quite slow because the network must be run separately for each patch, and there is a lot of redundancy due to overlapping patches. Secondly, there is a trade-off between localization accuracy and the use of context. Larger patches require more max-pooling layers that reduce the localization accuracy, while small patches allow the network to see only little context.

context는 이미지 내에서 객체를 인식하기 위해 필요한 주변 정보(배경, 주변 객체 등)를 의미한다. 즉, context가 충분히 확보되지 않으면, 모델이 객체를 정확하게 인식하지 못할 수 있다.

> In this paper, we **build upon** a more elegant architecture, **the so-called** “fully convolutional network” [9].

> The main idea in[9] is to **supplement** a usual contracting network **by** successive layers, where pooling operators are replaced by upsampling operators. Hence, these layers increase the resolution of the output. In order to localize, high resolution features from the contracting path are combined with the upsampled output. A successive convolution layer can then learn to assemble a more precise output based on this information.

> One important modification in our architecture is that in the upsampling part we have also a large number of feature channels, which allow the network to propagate context information to higher resolution layers.

```python
Decoder: 1024 -> 512 -> 256 -> 128 -> 64 # Allow the network to propagate context information to higher resolution layers.
```
Contracting network(Encoder)에서 추출한 feature map에는 context 정보가 압축되어 있는데, 이를 제대로 활용하기 위해서 upsampling part(Decoder)에 feature channel을 많이 두어서 context 정보를 높은 해상도 레이어로 전달할 수 있도록 한 것이 UNet의 중요한 특징 중 하나이다.

**지금 보면 여기서 Decoder에 Feature Channel을 많이 두어서 Context 확보를 충분히 했음을 알 수 있음.**

> This strategy allows the seamless segmentation of arbitrarily large images by an overlap-tile strategy (see Figure 2). To predict the pixels in the border region of the image, the missing context is extrapolated by mirroring the input image. This tiling strategy is important to apply the network to large images, since otherwise the resolution would be limited by the GPU memory.

U-Net은 패딩 없이 합성곱을 수행하는 Valid Convolution을 사용하기 때문에, 연산 과정에서 이미지 가장자리 픽셀이 점진적으로 잘려나간다. (input 층: 572x572 -> 570x570 -> 568x568)

 이로 인해 입력보다 출력 크기가 작아지는데, 특히 이미지 경계 근처 픽셀을 예측하려면 경계 너머의 맥락 정보가 필요하지만 실제로는 존재하지 않는다. 이를 보완하기 위해 원본 이미지 전체를 사방으로 mirroring하여 확장한 뒤 타일을 잘라 입력으로 사용함으로써, 경계 픽셀에 대해서도 충분한 주변 맥락을 확보할 수 있다.

> Valid Convolution이란 패딩을 사용하지 않고 입력 이미지의 경계 부분을 처리하는 합성곱 연산으로, 결과적으로 출력 크기가 입력 크기보다 작아지는 특성을 가진다.
